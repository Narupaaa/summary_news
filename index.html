⚙️ Obsidian News Vault Architecture

ระบบจัดการความรู้และวิเคราะห์การลงทุนอัตโนมัติ (Automated Investment Knowledge Management)

🔄 1. Data Flow (ขั้นตอนการทำงาน)

📥 Input (Raw)

รับข่าวดิบจากเว็บ, ลิงก์, หรือข้อความแชท วางในโฟลเดอร์ /Raw

🧠 Processing

Gemini CLI วิเคราะห์ Sentiment, สกัด Entity และเชื่อมโยง [[Link]]

📝 Documentation

สร้าง Daily Note และอัปเดตไฟล์ราย Topic ตามมาตรฐาน SMA 200

📊 Reporting

สร้าง Dashboard และ Intelligence Report สรุปภาพรวมรายเดือน

🚀 เปิดดูรายงาน Intelligence Report ล่าสุด

📂 Folder Structure

  • Daily/: บันทึกรายวัน (Timeline ของเหตุการณ์)
  • Topics/: ฐานข้อมูลรายสินทรัพย์ (Atomic Notes)
  • MOCs/: ศูนย์รวมลิงก์ (Maps of Content) แบ่งตามหมวดหมู่
  • Monthly/: รายงานสรุปผลงานรายเดือน

🛠️ Automation Scripts

  • generate_monthly_report.js: รวบรวมข้อมูล MD เป็น HTML
  • generate_demo.js: อัปเดตราคาและสัญญาณเทคนิคล่าสุด
  • export_all_md.js: รวมรวบฐานข้อมูลทั้งหมดเพื่อการวิเคราะห์

📈 วิสัยทัศน์: SMA 200 Methodology

เราใช้เส้นค่าเฉลี่ย 200 วันเป็นเกณฑ์หลักในการคัดกรองสินทรัพย์เพื่อลดสัญญาณรบกวน (Noise) ของตลาด:

🟢 Bullish (Above SMA 200)

สินทรัพย์อยู่ในแนวโน้มขาขึ้น เน้นการถือครองหรือหาจังหวะ Buy on Dip

🔴 Bearish (Below SMA 200)

สินทรัพย์อยู่ในแนวโน้มขาลง เน้นการเฝ้าระวัง หรือมองหา "จุดกลับตัว" (Reversal)

🟡 Pivot (At SMA 200)

จุดตัดสินใจสำคัญ (Decision Point) ที่มักเกิดการเปลี่ยนแนวโน้ม

🔗 ความสัมพันธ์แบบ Networked Thought

ระบบไม่ได้เก็บข้อมูลแยกจากกัน แต่เชื่อมโยงเป็นเครือข่าย (Graph View):

  • Hub & Spoke: MOC ทำหน้าที่เป็น Hub เชื่อมไปยัง Topics (Spoke)
  • Backlinking: ทุกข่าวใน Daily Note จะ Link ไปหา Topic เสมอ ทำให้เห็น Timeline การสะสมของข่าว
  • Ticker-Driven: ใช้ Metadata แบบมาตรฐานเพื่อให้ Script สามารถดึงราคาจริงมาวิเคราะห์ได้
